# Classes & Object-Oriented Programming (OOP)

## What Is OOP?

Object-Oriented Programming (OOP) is a way of organizing code around "objects" — bundles of related data and behavior. Instead of writing loose functions and variables, you group them into classes that represent real-world concepts.

Think of a class as a blueprint, and an object as something built from that blueprint. A `Car` class is the blueprint; a specific car (red, 2020, Toyota) is the object.

---

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Class** | A blueprint for creating objects with shared properties and methods |
| **Constructor** | A special method that runs when you create a new object from a class |
| **Properties** | Data stored inside an object (like `car.color`) |
| **Methods** | Functions that belong to an object (like `car.start()`) |
| **`this`** | A keyword that refers to the current object |
| **Inheritance** | Creating a new class based on an existing class |
| **Encapsulation** | Hiding internal details and only exposing what is necessary |
